# ТЗ: Telegram-бот для сбора еженедельных заказов MiniMelts

## 1. Общее описание

Telegram-бот для сбора заказов от постоянных клиентов (до 50-100 человек).
Ассортимент — мороженое MiniMelts, до 20 наименований.
Заказы собираются 1-2 раза в неделю.

**Принцип:** максимальная простота. Клиент открывает бот, вводит количество по каждой позиции, отправляет заказ. Всё.

---

## 2. Роли

| Роль | Описание |
|------|----------|
| **Клиент** | Постоянный покупатель. Видит каталог, вводит количество, отправляет заказ. Видит ТОЛЬКО свои заказы. |
| **Админ** | Владелец бизнеса. Управляет каталогом, видит все заказы, открывает/закрывает приём, выгружает сводку. |

---

## 3. Сценарии клиента

### 3.1 Первый запуск (/start)
1. Бот просит представиться (имя + название точки/компании)
2. Клиент вводит данные
3. Бот сохраняет клиента со статусом "ожидает подтверждения"
4. Админ получает уведомление о новом клиенте и подтверждает/отклоняет

### 3.2 Оформление заказа (кнопка "Новый заказ")
1. Клиент нажимает "Новый заказ"
2. Бот проверяет: открыт ли сейчас приём заказов?
   - Если нет — сообщение "Приём заказов пока закрыт, ждите уведомления"
   - Если да — переход к шагу 3
3. Бот последовательно показывает КАЖДУЮ позицию каталога:
   ```
   🧊 Cookies & Cream
   Введите количество (0 если не нужно):
   ```
4. Клиент вводит число (0 — пропустить)
5. После всех позиций — показывается итог заказа:
   ```
   Ваш заказ:
   • Cookies & Cream — 5
   • Vanilla — 10
   • Mango Sorbet — 3

   Итого позиций: 3, единиц: 18

   [Подтвердить] [Отменить] [Изменить]
   ```
6. "Подтвердить" — заказ сохраняется, админ получает уведомление
7. "Изменить" — возврат к вводу количества (заново по позициям)
8. "Отменить" — заказ удаляется

### 3.3 Мои заказы
- Клиент видит свои последние заказы (дата, состав, статус)
- Другие клиенты ничего чужого НЕ видят

## 4. Сценарии админа

### 4.1 Управление приёмом заказов
- **"Открыть приём"** — бот рассылает всем клиентам уведомление:
  ```
  📢 Приём заказов открыт! Сделайте заказ до [дата/время].
  ```
- **"Закрыть приём"** — заказы больше не принимаются

### 4.2 Сводка заказов
- **"Сводка"** — бот формирует таблицу:
  ```
  📊 Сводка заказов от 18.02.2026

  Всего заказов: 35 из 50 клиентов

  Товар               | Всего
  --------------------|------
  Cookies & Cream     | 120
  Vanilla             | 95
  Mango Sorbet        | 45
  ...

  ИТОГО единиц: 580
  ```
- **"Выгрузка Excel"** — бот отправляет .xlsx файл с детализацией:
  - Строки = клиенты
  - Столбцы = товары
  - Ячейки = количество

### 4.3 Управление каталогом
- **"Каталог"** — список позиций с кнопками:
  - Добавить позицию (название)
  - Удалить позицию
  - Изменить порядок (вверх/вниз)

### 4.4 Управление клиентами
- **"Клиенты"** — список с возможностью:
  - Подтвердить нового клиента
  - Заблокировать клиента
  - Посмотреть историю заказов клиента

### 4.5 Ручное напоминание
- **"Напомнить"** — бот рассылает клиентам, которые ещё НЕ сделали заказ:
  ```
  ⏰ Напоминание: приём заказов открыт. Не забудьте оформить заказ!
  ```

### 4.6 Автоматические напоминания по расписанию
Бот сам пушит клиентов по расписанию, без участия админа:

- **Автооткрытие приёма** — в заданные дни недели (например, Пн и Чт в 09:00) бот автоматически открывает сессию и рассылает всем клиентам:
  ```
  📢 Доброе утро! Приём заказов открыт. Сделайте заказ до среды 20:00.
  ```
- **Автонапоминание** — за N часов до дедлайна бот напоминает тем, кто ещё НЕ заказал:
  ```
  ⏰ Приём заказов закрывается сегодня в 20:00. Вы ещё не оформили заказ!
  ```
- **Автозакрытие** — по дедлайну бот автоматически закрывает приём и присылает админу сводку

**Настройки расписания (в .env):**
- Дни открытия приёма (день недели + время)
- Дедлайн закрытия (день недели + время)
- За сколько часов до дедлайна отправлять напоминание

Админ может в любой момент переопределить расписание вручную (открыть/закрыть приём досрочно).

## 5. Технический стек

| Компонент | Технология |
|-----------|-----------|
| Язык | Python 3.11+ |
| Telegram API | aiogram 3.x |
| База данных | SQLite (через aiosqlite) |
| Excel-выгрузка | openpyxl |
| Планировщик | APScheduler (для автонапоминаний по расписанию) |
| Конфигурация | .env файл (токен бота, ID админа, расписание) |

### Почему этот стек:
- **Python + aiogram** — самый популярный стек для Telegram-ботов, куча документации
- **SQLite** — для 50-100 клиентов сервер БД избыточен, файловая БД идеальна
- **Простота деплоя** — один процесс, один файл БД, запуск через systemd или Docker

## 7. Схема базы данных

```sql
-- Клиенты
CREATE TABLE clients (
    id INTEGER PRIMARY KEY,
    telegram_id INTEGER UNIQUE NOT NULL,
    name TEXT NOT NULL,
    company TEXT,
    status TEXT DEFAULT 'pending',  -- pending / approved / blocked
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Каталог товаров
CREATE TABLE products (
    id INTEGER PRIMARY KEY,
    name TEXT NOT NULL,
    sort_order INTEGER DEFAULT 0,
    is_active INTEGER DEFAULT 1
);

-- Сессии приёма заказов
CREATE TABLE order_sessions (
    id INTEGER PRIMARY KEY,
    opened_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    closed_at TIMESTAMP,
    is_active INTEGER DEFAULT 1
);

-- Заказы
CREATE TABLE orders (
    id INTEGER PRIMARY KEY,
    client_id INTEGER REFERENCES clients(id),
    session_id INTEGER REFERENCES order_sessions(id),
    status TEXT DEFAULT 'confirmed',  -- confirmed / cancelled
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Позиции заказа
CREATE TABLE order_items (
    id INTEGER PRIMARY KEY,
    order_id INTEGER REFERENCES orders(id),
    product_id INTEGER REFERENCES products(id),
    quantity INTEGER NOT NULL DEFAULT 0
);
```

## 8. Конфигурация (.env)

```
BOT_TOKEN=123456:ABC-DEF...
ADMIN_IDS=123456789,987654321

# Расписание (cron-формат: день_недели час:минуты)
# Дни недели: MON, TUE, WED, THU, FRI, SAT, SUN
SCHEDULE_OPEN=MON:09:00,THU:09:00       # Когда открывать приём
SCHEDULE_CLOSE=WED:20:00,SAT:20:00      # Когда закрывать приём (дедлайн)
REMINDER_HOURS_BEFORE=4                   # За сколько часов до дедлайна напоминать
TIMEZONE=Europe/Madrid                    # Часовой пояс
```

- `ADMIN_IDS` — через запятую, можно несколько админов
- Расписание можно не задавать — тогда работает только ручное управление
